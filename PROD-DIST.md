# PROD-DIST.md — Production Build Guide

This project ships as a **single Spring Boot JAR** that includes the compiled Angular frontend as static files.

## Prerequisites

- Java 21
- Node 20+ / npm
- Maven

## Generate the production build

From the **project root**, run:

```bash
./build-dist.sh
```

The script does the following in order:

1. Runs `npm ci && npm run build` inside `frontend/` — compiles Angular into `frontend/dist/frontend/browser/`
2. Clears `backend/src/main/resources/static/` and copies the Angular output there
3. Runs `mvn clean package -DskipTests` inside `backend/` — packages everything into a single JAR

The final artifact is:

```
backend/target/swe-quizzes-0.0.1-SNAPSHOT.jar
```

## Run the JAR locally

```bash
java -jar backend/target/swe-quizzes-0.0.1-SNAPSHOT.jar \
  --spring.datasource.url=jdbc:postgresql://localhost:5432/swequizzes \
  --spring.datasource.username=swequizzes \
  --spring.datasource.password=swequizzes
```

App is available at `http://localhost:8080`.

## Notes

- The Angular app is served from `/` by Spring Boot's static resource handler.
- All `/api/**` requests are handled by Spring Boot controllers.
- Deep links (e.g. `/category/aws`) are forwarded to `index.html` by the `SpaController`.
- Do **not** commit the `backend/src/main/resources/static/` folder — it is generated by the build script.
